# 技术术语表（持续更新）

## JSON
- **是什么**：JavaScript对象表示法，一种轻量级数据交换格式。
- **为什么用**：机器易读易写，适合程序间传递结构化数据。
- **怎么用最好**：字段名清晰，避免嵌套过深，必要时添加版本号。

## 命令行参数
- **是什么**：在终端运行程序时附加的额外信息，通过 `sys.argv` 获取。
- **为什么用**：无需修改代码即可改变程序行为，适合快速测试。
- **怎么用最好**：提供默认值，参数顺序固定，支持 `-h` 帮助信息。

## API
- **是什么**：应用程序编程接口，让不同软件之间可以通信。
- **为什么用**：调用他人提供的功能（如AI模型），无需自己实现。
- **怎么用最好**：妥善保管密钥，处理错误响应，设置超时时间。

## SSL证书
- **是什么**：安全套接层证书，用于验证网站身份并加密通信。
- **为什么用**：防止中间人攻击，确保数据传输安全。
- **怎么用最好**：保持系统根证书更新；在国内网络下可临时关闭验证（`http.sslVerify false`）以解决连接问题，但需注意安全风险。

## 正则表达式
- **是什么**：一种描述字符串模式的工具，可用于搜索、替换、提取文本。
- **为什么用**：处理非结构化文本（如AI返回内容）时非常强大。
- **怎么用最好**：先测试再使用，避免过于复杂导致难以维护。

## 个人访问令牌
- **是什么**：GitHub提供的一种替代密码的认证方式，可设置权限和有效期。
- **为什么用**：比密码更安全，可随时撤销，适合用于程序自动操作。
- **怎么用最好**：只授予最小必要权限（如 `repo`），妥善保管，定期更换。

## MVP
- **是什么**：最小可行产品（Minimum Viable Product），指刚好能验证核心价值假设的最简产品版本。
- **为什么用**：用最小成本最快速度验证市场，避免浪费资源开发无人需要的功能。
- **怎么用最好**：聚焦核心场景，忽略边缘细节，尽早获取用户反馈。

## 按次付费
- **是什么**：一种收费模式，用户每次使用服务支付一次费用。
- **为什么用**：降低低频用户的决策门槛，适合需求不频繁的场景。
- **怎么用最好**：搭配套餐和订阅，形成完整付费体系；明确定义“一次”的边界。

## 订阅制
- **是什么**：用户定期（如每月/每年）支付固定费用，获得持续服务。
- **为什么用**：收入稳定，用户粘性高，适合高频使用的场景。
- **怎么用最好**：提供分层定价（基础版/专业版/企业版），满足不同需求。
